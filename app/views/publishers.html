Data publishers prototypes

PP 2  Publish manually or automaticallyALPHA

June 2017

Main changes
Change Why we made it
Test a new option to explore automatic publishing of data via APIs. We've found from users that automatic publishing is the ideal scenario, but hard to get set up for certain services/organisations. We think that offering a more hands-on service to get users started will make this easier.
Group the previously tested ideas into one, with two routes for different responsibilities. To test if users knew whether they were responsible for one or more services. We also need to test the previous prototypes with more users who publish data for multiple services so we can be sure of our findings.

PP 1  Publish for services or departmentsALPHA

June 2017

Two options to test different levels of responsibility:

  • One person responsible for one service, with ability to delegate responsibility for individual metrics
  • One person responsible for one department, with ability to delegate responsibility for individual services
