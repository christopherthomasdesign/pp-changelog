Performance Platform changelog

All prototypes created during the development of the Performance Platform since the re-discovery in early 2016.

If you haven't been given access to these, you'll need to get in touch with the team.

PP–3 ALPHA

November 2016

Changes made
Change Why we made it
Did a thing Because we thought it would be cool

PP–2 ALPHA

October 2016

Changes made
Change Why we made it
Add a search function with autocomplete to the home page We found in research that most participants wanted to be able search for something specific. We think an autocomplete will help suggest what it's possible to search for
Added user effort data, broken down by 'reasons users get in touch' and 'reasons users have to repeat or amend something' To test whether this kind of information is useful for our users to assess service quality
Removed transactional service ('task') pages To test our hypothesis that our users only want to see the data at a higher level (i.e. department, agency and so on)

PP–1 DISCO ALPHA

August 2016

Changes made
Change Why we made it
Create a clickable interface for browsing the data We found that the downloadable spreadsheet in the last iteration was too overwhelming for most users (although some need the ability to download in that format too)
Show aggregated data by department, agency, topic and 'sub-topic' We think that grouping data in different ways will allow our users to get a higher-level view of things across government
Add a filter/comparison tool We think that users may need to create their own views of the data by combining different departments or areas of interest

PP–0 DISCOVERY

April 2016

Spreadsheet to test the different data types we need to collect.
